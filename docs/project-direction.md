# 项目方向

## 项目定义

本项目要做的是一个每天自动更新的网页日报站点，用来汇总和展示枝江娱乐旗下虚拟艺人的前一日直播相关信息。

当前设想是：

- 每天早上 `08:00` 自动更新当日日报页面
- 以公开网页形式呈现，而不是只做后台工具
- 使用真实域名对外访问

## 域名规划

当前已购买并确定使用的正式域名为 `zhijiangdaily.com`。

说明：

- 第一版上线优先绑定主域名 `zhijiangdaily.com`
- 后续可再决定是否同时接入 `www.zhijiangdaily.com`

## 内容对象

日报聚焦于枝江娱乐旗下两个团体的艺人动态：

### A-SOUL

- 嘉然
- 乃琳
- 贝拉

### 枝江二期闪耀舞台

- 心宜
- 思诺

## 核心内容板块

现阶段日报最主要的内容板块如下，其他小功能以后再扩展：

1. 每日直播日程表
2. 昨日直播数据
3. 昨日直播高播放切片
4. 昨日直播热点总结与反馈

## 内容来源原则

“昨日直播热点总结与反馈” 这一板块，当前定义为主要基于以下来源进行整理：

- 成员官号内容
- 直播切片评论区反馈

后续如果增加更多数据源，也应该先在这里补充定义。

## 产品目标

把“枝江日报”做成一个稳定的日报生产与展示站点：

- 前台负责清晰展示每日内容
- 系统每天自动生成或更新日报
- 后台负责抓取、整理、审核和发布
- 整体流程要围绕“稳定产出日报”持续增强

## 当前阶段

当前重点是先把第一版上线链路打稳：

- 首先完成 `zhijiangdaily.com` 的第一版上线
- 第一版允许以当前 localhost 草稿内容作为展示基础
- 先确保“能打开、能预览、能人工发布”
- 再逐步接入更完整的自动抓取和定时更新

## 当前优先级

1. 完成第一版 Vercel 部署
2. 完成 `zhijiangdaily.com` 域名绑定
3. 确保首页和单期日报页可正常访问
4. 稳定抓取、生成、发布这条主流程
5. 再落实每天 `08:00` 自动更新机制

## 第一版上线范围

第一版不追求全部自动化，优先保证：

- 站点已上线并可访问
- 使用现有草稿内容作为基础展示
- 后台可以继续手动触发抓取、生成、编辑、发布
- 数据源先覆盖已确认的官方动态来源

## 暂不优先

- 复杂社区功能
- 非核心互动玩法
- 多实例部署
- 重度运营后台

## 产品原则

- 前台内容质量优先于后台复杂功能
- 核心板块先做稳定，再做丰富
- 自动生成结果必须允许人工校正
- 数据结构宁可清晰，也不要过度抽象
- 任何新增功能都不应干扰日报主链路

## 技术原则

- 页面层尽量少写业务逻辑
- 业务逻辑优先沉到 `lib/`
- 静态资源统一放在 `public/`
- 产品定义与范围以本文档为准

## 后续维护规则

以下变化发生时，应优先更新本文件：

- 正式域名变更
- 关注艺人名单调整
- 核心板块变化
- 自动更新时间变化
- 产品目标或范围变化

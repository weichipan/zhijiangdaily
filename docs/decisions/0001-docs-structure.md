# 0001 文档结构分层

## 背景

项目协作逐渐依赖对话上下文，容易在上下文截断后重复讨论和重复实现。

## 决策

采用分层文档结构：

- `README.md` 负责项目入口
- `AGENTS.md` 负责协作约定
- `docs/project-direction.md` 负责方向与范围
- `docs/architecture.md` 负责结构与职责
- `docs/context-summary.md` 负责阶段总结
- `docs/decisions/` 负责长期决策记录

## 原因

- 入口信息和长期规则不应混在同一处
- 方向、结构、决策、阶段状态的更新频率不同
- 这样能减少对一次聊天上下文的依赖

## 影响

- 后续新增长期约定时，需要同步更新对应文档
- 结构性改动后，应补阶段总结或决策记录

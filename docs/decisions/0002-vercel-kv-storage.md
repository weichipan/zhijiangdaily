# 0002 Vercel 存储切换策略

## 背景

项目最初把运行数据直接写入 `data/*.json`。这种方式适合本地开发，但不适合部署到 Vercel，因为 Vercel 的运行环境不提供可依赖的持久本地文件系统。

## 决策

采用“双模式存储”：

- 本地开发默认使用 `data/*.json`
- 当环境变量中存在 `KV_REST_API_URL` 和 `KV_REST_API_TOKEN` 时，自动切换到 Vercel KV

## 原因

- 能保留当前本地开发和调试体验
- 不需要把整个项目一次性重构到数据库 ORM
- 上层页面和 API 可以继续通过 `lib/repository.js` 访问数据，改动面较小

## 影响

- Vercel 部署不再依赖本地文件写入
- 首页、详情页和后台需要保持动态渲染
- 运行环境需要额外配置 KV 环境变量

## 后续动作

- 在真实 Vercel 项目中完成 KV 绑定并验证写入
- 评估后续是否迁移到 PostgreSQL 以支持更复杂查询与报表
